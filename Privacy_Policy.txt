Privacy Policy
Extension: Epstein & Associates Blocker
Effective date: 5 May 2026
Last updated: 5 May 2026

Summary: This extension does not collect, transmit, sell, or share any personal data or browsing information. All processing happens locally on your device. No data ever leaves your browser.
1. Overview
Epstein & Associates Blocker ("the Extension") is a Google Chrome extension that intercepts navigations to websites and web pages containing content related to individuals named in the Epstein court documents. When a match is detected, the Extension redirects the browser to a locally hosted block page. Users may choose to allow access to any blocked site for the remainder of their browser session.

This Privacy Policy explains what information the Extension accesses, how that information is used, and confirms that no personal data is collected or transmitted at any time.

2. Data We Do Not Collect
The Extension does not collect, record, store externally, or transmit any of the following:

Your browsing history or the URLs of pages you visit
The content of any web page you view
Personal identifiers of any kind (name, email address, IP address, device ID, etc.)
Cookies, authentication tokens, or session data from any website
Search queries or form input
Crash reports, diagnostic data, or usage analytics
3. What the Extension Does Access (Locally Only)
To perform its blocking function, the Extension operates entirely within your browser:

3a. Page Text Content
The Extension's content script reads the rendered text of each web page you visit (document.body.textContent) and tests it against a locally stored list of names using a regular expression. This check happens entirely in your browser's memory. The text is never stored, logged, or sent anywhere — it is read once and immediately discarded.

3b. Navigation URLs (Network Level)
The Extension registers network-level rules using Chrome's declarativeNetRequest API. These rules are evaluated by the Chrome browser engine itself — not by any external server — to decide whether a navigation should be redirected. The Extension's code does not receive or log the URLs of pages you visit; the rules operate as a filter applied by Chrome before the request is made.

3c. Locally Stored Allow List
When you click "Allow This Site" on the block page, the domain name of the site you chose to allow (for example, microsoft.com) is saved to chrome.storage.session. This is a temporary storage area built into Chrome with the following properties:

It is stored only on your local device, never on any server.
It is automatically and permanently deleted when you close your browser.
It contains only the plain domain name (e.g. microsoft.com), not any URLs, page content, or personal information.
It is accessible only to this Extension and to no other website or service.
4. Data Sharing and Third Parties
The Extension does not share any data with any third party. It makes no network requests of its own — it does not contact any external server, analytics platform, advertising network, or any other remote service. There are no third-party libraries, SDKs, or tracking tools embedded in the Extension.

5. Remote Code
The Extension does not download or execute any code from a remote server. All JavaScript that runs in the Extension is bundled within the Extension package and served locally from the chrome-extension:// origin. This is fully compliant with Google's Manifest V3 policy prohibiting remotely hosted code.

6. Permissions Explanation
The Extension requests the following Chrome permissions:

declarativeNetRequest — Required to register the network-level rules that redirect navigations to blocked domains and URLs to the local block page, and to add allow rules when the user permits a site.
storage — Required to temporarily save the list of domains the user has explicitly allowed for the current browser session, so that the content script knows not to re-block those sites. This data is session-only and never leaves the device.
Host permission (<all_urls>) — Required because the Extension cannot predict in advance which websites may contain content related to the individuals on the blocklist. The content script must be able to check page text on any domain. No browsing data from these sites is collected or transmitted.
7. Children's Privacy
The Extension does not collect personal information from anyone, including children under the age of 13. No age-gated content or age verification is required because no personal data is collected at any point.

8. Your Control Over the Extension
You have full control over the Extension at all times:

Allow a site temporarily: Click "Allow This Site" on the block page. Access is granted for the current browser session only and is revoked automatically when you close Chrome.
Disable the Extension: Open chrome://extensions and toggle the Extension off. No data persists after the browser session ends regardless.
Uninstall the Extension: Open chrome://extensions, find the Extension, and click "Remove". All local session data (if any) is permanently deleted upon removal.
